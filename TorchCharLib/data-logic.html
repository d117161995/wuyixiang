🚦 角色筛选 / 过滤逻辑

角色列表直接来源于 CGTeamwork，不依赖本地目录是否存在。需要同时满足以下条件：

  1. 条件1 角色在 CGTeamwork proj_film.m_asset（type=Cha）中存在
  2. 条件2 · 核心 CGTW 自定义备注（m_asset.notes）不为空 — notes 为空的角色直接剔除，不论本地目录是否存在、是否有资产文件
  3. 条件3 CGTW 资产状态（m_asset.status）不为 Close
关键规则： notes 为空 = 不收录。即使本地 X:\...\Cha\{角色} 目录已存在且有完整资产，只要 CGTW 中 notes 字段为空，该角色就不会出现在数据库中。

满足条件的角色全部进入数据库。若本地 X:\projects2024\TLI\Asset\Cha\{角色} 目录存在，则扫描其资产文件；若不存在，角色仍保留在库中（资产模块显示"暂无资源"）。

notes 格式： ss10_英雄pv|宾|Bin|主角
四段含义：首次出现集数 | 中文名 | 英文名 | 角色等级（主角/配角）

排序规则（主角/配角各自独立排序）

侧边栏角色排序分为主角组和配角组，每组内的排序逻辑相同：

  1. 第一级 按 first_appear（首次出现集数）从大到小排序。集数格式如 ss12_2 > ss12_1 > ss11_2 > ss11_1 > ss10 > ss9
  2. 第二级 同一集数内，按 create_time（角色创建时间）从新到旧排序
  3. 第三级 · 同前缀聚合 同一集数内，如果多个角色的命名前缀相同（如 Mouse 系列的 MouseC / MouseD / MouseF），则将它们聚合在一起，插入到该前缀中创建时间最新的那个角色的位置
前缀提取规则：从 ID 中提取字母部分，去掉末尾的单个大写字母后缀和数字后缀。
例：c028_MaoB → mao，c013_Erica3 → erica，c019_MouseC → mouse

聚合示例：假设时间排序本应为 猪、猫03、狗、猫02、猫01 → 聚合后变为 猪、猫03、猫02、猫01、狗。同前缀的角色"吸附"到该前缀首次出现的位置。

🔄 数据更新流程

更新数据库分两步执行，需在项目根目录下运行：

1 拉取 CGTeamwork 数据

从 CGTeamwork 获取所有 Cha 资产的中文名、notes 备注、status 状态、首次出现集数、创建时间和缩略图。

# 登录 CGTW → 查询资产 → 下载缩略图 → 输出 cgtw-asset-names.json
node TorchCharLib/fetch-cgtw-names.mjs

输出文件：

  • cgtw-asset-names.json — CGTW 资产元数据（含 cn_name, notes, status, first_appear, create_time, thumbnail）
  • thumbnails/ — 下载的缩略图文件夹

2 扫描本地资产目录

扫描 X:\projects2024\TLI\Asset\Cha，结合 CGTW 白名单过滤，生成最终数据库。

# 读 cgtw-asset-names.json → 扫描本地文件 → 过滤 → 输出 JSON
node TorchCharLib/scan-char-assets.mjs

输出文件：

  • torch-char-db.json — 完整角色数据库（前端 fetch 用）
  • torch-char-db.data.js — 内联 JS 数据（作为 window.__TORCH_CHAR_DB__）

完整更新流水线

CGTeamwork API → cgtw-asset-names.json → 过滤 (notes + status) → 本地资产扫描（若目录存在） → torch-char-db.json → 网页渲染
# 一键更新（两步合一）
node TorchCharLib/fetch-cgtw-names.mjs
node TorchCharLib/scan-char-assets.mjs
注意： 执行脚本需要能访问 X: 盘（资产服务器）和 CGTeamwork 内网。更新后刷新浏览器即可看到最新数据（已配置 Cache-Control: no-cache）。
🌐 网页端数据加载与展示

数据加载

页面加载时优先通过 fetch 从服务器获取 torch-char-db.json（带时间戳和 cache: "no-store" 绕过缓存）。如果 fetch 失败，回退到内联的 torch-char-db.data.js（window.__TORCH_CHAR_DB__）。

刷新数据库按钮

点击页面上的"刷新数据库"按钮会重新 fetch 最新 JSON 并更新 UI，无需整页刷新。尝试保留当前选中的角色。

侧边栏展示

  • 角色显示名：notes_episode + notes_cn
  • 角色页面标题：notes_cn - notes_en (notes_role)
  • 分组：主角组 / 配角组（折叠面板）
  • 排序：按 JSON 数据中的顺序（已在生成时排好）

图片展示规则（2D游戏设计 / 2D影视设计模块）

  • 1张图：填满模块 100% 宽高，object-fit: contain
  • 2张图：各占 50% 宽、100% 高，object-fit: contain
  • 3-4张图：2×2 网格，各占 50% 宽高，object-fit: contain
  • 5张及以上：同 2×2 网格，超出部分滚动

模块固定高度 400px，图片保持原始比例不裁切。

视觉布局

右侧资产展示分为上下两部分：

  • 上部：左右结构 — 左侧竖排"2D游戏设计"和"2D影视设计"，右侧"3D影视设计（LookDev）"占满左侧两个模块的总高度
  • 下部：左右各半 — 左列"模型/贴图Maya/绑定"文件路径（可点击复制），右列内联 3D FBX 查看器
