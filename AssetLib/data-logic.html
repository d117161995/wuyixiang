统一数据流程

CGTeamwork API → cgtw-asset-names.json → 过滤 (notes + status) → 本地资产扫描（若目录存在） → {project}-char-db.json → 网页渲染

所有项目遵循同一套数据流水线：先从 CGTeamwork 拉取角色元数据，过滤后扫描本地资产目录，生成 JSON 数据库供前端展示。

统一角色筛选逻辑

角色列表直接来源于 CGTeamwork，不依赖本地目录是否存在。需同时满足：

  1. 条件1 角色在 CGTeamwork 对应数据库的 m_asset（type=Cha）中存在
  2. 条件2 · 核心 CGTW 自定义备注（m_asset.notes）不为空 — notes 为空的角色直接剔除，不论本地目录是否存在、是否有资产文件
  3. 条件3 CGTW 资产状态（m_asset.status）不为 Close
关键规则： notes 为空 = 不收录。即使本地资产目录已存在且有完整资产，只要 CGTW 中 notes 字段为空，该角色就不会出现在数据库中。

满足条件的角色全部进入数据库。若本地目录不存在，角色仍保留（资产模块显示"暂无资源"）。

notes 格式： 集数|中文名|英文名|角色等级
示例：ss10_英雄pv|宾|Bin|主角

统一排序规则（主角/配角各自独立排序）

  1. 第一级 按 first_appear（首次出现集数）从大到小排序
  2. 第二级 同一集数内，按 create_time（角色创建时间）从新到旧排序
  3. 第三级 同一集数内，相同命名前缀的角色聚合在一起
前缀提取规则：从 ID 中提取字母部分，去掉末尾的单个大写字母后缀和数字后缀。
例：c028_MaoB → mao，c013_Erica3 → erica
